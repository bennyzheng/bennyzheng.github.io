如何降低try-catch对性能的影响

发表于 2016年12月14日 | 分类于 javascript |

一直都知道try-catch对代码运行速度会有影响，今天刚好有这方面的尝试，于是试验了一下，试验场景是：

  1. 不使用try-catch
  2. try-catch内部费时代码是否放到函数中时性能区别
    • 直接将费时代码放在try-catch中
    • 将费时代码放入内置函数中
    • 将费时代码放到外部函数中进行调用
  3. try-catch内部是否引用外部变量时性能区别
    • 引用外部变量
    • 不引用外部变量
  4. try-catch是否放在子作用域时性能区别
    • 放在子作用域
    • 不放在子作用域

在我的机子上运行多次后得出类似这样的结果：

基本上可以得出初步结论：

  1. 尽量不使用try-catch，性能确实影响相当大
  2. try-catch内部代码尽可能使用外部函数调用
  3. try-catch应该使用函数包起来，对性能提升明显
